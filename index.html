This app is originally made for tracking a garment production with a simple flow. Well there were barely a certain flow actually. This app is such a mess currently and could be much more because of the uncertain flow.

This documentation could may help you on developing and understanding such a mess. Please go through the documentation carefully to know how this messed up flow was made and some advices for developing.

For this project you should at least understand how to use PHP with Laravel Framework, Livewire, Redis, MySQL Query Building with NaviCat, styling with a library such as Bootstrap etc. with an understanding of vanilla CSS as well, JavaScript and JQuery . You may need these documentations, Laravel Documentation, Livewire Documentation, Bootstrap Documentation.

In this project circumstances the most thing you may need is an excellent communication with your partners. You could learn so much from your partners. Because there are so many abnormal cases in this project.

Services Brief Review

PHP

This project currently using PHP v7.3^ You could use a tools like XAMPP/Laragon or you could install the apache services your way, it's up to you. The way this app using PHP wasn't that advance. It's just basic PHP with the help of Laravel Framework but it still messy, so much works needed to organize this app. You may need some custom configuration on php.ini you should've knew it.

MariaDB / MySQL

This project currently using MariaDB v10.4.27 You could use a tools like XAMPP/Laragon as well as PHP. The use of Query in this project is quite extreme. Especially on Report. For database management and query testing using NaviCat software. Raw Query will often be found in this project. You may need some custom configuration too on my.ini

Laravel

Currently using Laravel 8 with basic MVC usage. With Model for Table Access and Relationship, View for Client View and Controller for Business Logic. This project using Livewire, it's important to learn livewire events because the current usage of it is kinda mess, kinda complicated, it needs a good understanding of Javascript for developing this project's livewire components. One more thing that should be known is the usage of Laravel Excel, it's quite intermediate, the example is the usage of Charts on export excel, you will need to read the documentation, it could help you.

Javascript

The usage of Javascript in this project is quite complex. Currently put in the blade views. Javascript have an important role in this project. It usually use JQuery Ajax for backend request. There are lots of logics that using Javascript. It use a lot of library for general, like Datatable, Select2, SweetAlert, Izitoast. You need a good understanding of Javascript, for sure.

CSS

It's just basic styling with CSS in this project. Using Bootstrap 5 generally. It may need a lot of adjustments, but it could wait. Not something you need to worry.

Model

Here is the example of a Model on this project

                                    namespace App\Models\SignalBit;

                                    use Illuminate\Database\Eloquent\Factories\HasFactory;
                                    use Illuminate\Database\Eloquent\Model;

                                    class MasterPlan extends Model
                                    {
                                        use HasFactory;

                                        protected $connection = 'mysql_sb';

                                        protected $table = 'master_plan';

                                        protected $guarded = [];

                                        public $timestamps = false;

                                        public function userPassword()
                                        {
                                            return $this->belongsTo(UserLine::class, 'sewing_line', 'username');
                                        }

                                        public function actCosting()
                                        {
                                            return $this->belongsTo(ActCosting::class, 'id_ws', 'id');
                                        }

                                        // QC Endline
                                        public function rfts()
                                        {
                                            return $this->hasMany(Rft::class, 'master_plan_id', 'id');
                                        }

                                        public function defects()
                                        {
                                            return $this->hasMany(Defect::class, 'master_plan_id', 'id');
                                        }

                                        public function rejects()
                                        {
                                            return $this->hasMany(Reject::class, 'master_plan_id', 'id');
                                        }

                                        public function reworks()
                                        {
                                            return $this->hasMany(Rework::class, 'master_plan_id', 'id');
                                        }

                                        // QC Packing
                                        public function rftsPacking()
                                        {
                                            return $this->hasMany(OutputPacking::class, 'master_plan_id', 'id');
                                        }
                                    }

It use a middleware for this. You can see the User, Role & Access Table on Database Structure. The middleware looks like this :

                                    namespace App\Http\Middleware;

                                    use Closure;
                                    use Illuminate\Support\Facades\Auth;
                                    use Illuminate\Http\Request;

                                    class RoleMiddleWare
                                    {
                                        /**
                                        * Handle an incoming request.
                                        *
                                        * @param  \Illuminate\Http\Request  $request
                                        * @param  \Closure(\Illuminate\Http\Request): (\Illuminate\Http\Response|\Illuminate\Http\RedirectResponse)  $next
                                        * @return \Illuminate\Http\Response|\Illuminate\Http\RedirectResponse
                                        */
                                        public function handle(Request $request, Closure $next, ...$roles)
                                        {
                                            $user = Auth::user();

                                            if (in_array("superadmin", $roles)) {
                                                if ($user->roles->whereIn("nama_role", ["superadmin"])->count() > 0) {
                                                    return $next($request);
                                                }
                                            } else if (in_array("admin", $roles)) {
                                                if ($user->roles->whereIn("nama_role", ["admin", "superadmin"])->count() > 0) {
                                                    return $next($request);
                                                }
                                            } else {
                                                if (!(in_array("accounting", $roles)) && $user->roles->whereIn("nama_role", ["admin", "superadmin"])->count() > 0) {
                                                    return $next($request);
                                                }

                                                foreach($roles as $role) {
                                                    // Check if user has the role This check will depend on how your roles are set up
                                                    foreach ($user->roles as $userRole) {
                                                        if (($role == 'accounting' && $userRole->accesses->whereIn("access", [$role])->count() > 0) || ($role != 'accounting' && $userRole->accesses->whereIn("access", [$role, "all"])->count() > 0)) {
                                                            return $next($request);
                                                        }
                                                    }
                                                }
                                            }

                                            return redirect('home')->with('error', 'You have not access to this module');
                                        }
                                    }

                                        public $date;
                                        public $line;

                                        {
                                            $this->date = date('Y-m-d');

                                        {
